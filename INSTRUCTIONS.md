# 🛠 Руководство разработчика (Для людей и ИИ агентов)

Этот документ описывает внутреннюю логику работы скриптов и конвейера автоматизации. Прочитайте его, прежде чем вносить изменения в архитектуру проекта.

## ⛔ Внешние зависимости (Upstream)

Папок `Re-filter-lists` и `domain-list-community` в самом Git-репозитории **НЕТ** по причине того, что они занесены в `.gitignore`. 
- **На сервере (GitHub Actions):** они динамически клонируются из оригинальных репозиториев (`1andrevich/Re-filter-lists`, `v2fly/domain-list-community`) перед запуском Python скриптов.
- **Локально (у вас на ПК):** вы можете выкачать и обновить их, запустив скрипт `./pipeline.sh` (внутри него встроен `git pull` для этих папок).

## 🐛 Локальный Дебаг и Разработка

Вам **не нужно** каждый раз делать `git commit` и ждать, пока GitHub Actions отработает, чтобы проверить свои изменения. Проект оптимизирован для работы на локальной машине.

### Как вынести сервис в отдельный (альтернативный) маршрут?
1. Откройте файл `alternative.txt` в корне.
2. Впишите туда название категории из `domain-list-community` (Например: `telegram`, `youtube`, `google`). Каждое имя с новой строки.
3. **Виртуальные категории CIDR:** Скрипты также генерируют независимые подсети в виде виртуальных категорий (`telegram`, `discord`, `asn`). Вписав `telegram` в `alternative.txt`, вы автоматически перенесёте в альтернативный маршрут **и домены Telegram, и все его официальные IP-подсети**. То же касается голосовых узлов `discord`. Сети 16-Кб блокировок живут под виртуальной категорией `asn` (по умолчанию они всегда попадают в основной `filtered` маршрут).

### Как полностью удалить сервис из всех списков (настоящий exclude)?
1. Откройте файл `exclude.txt` в корне.
2. Впишите название категории (например: `microsoft`), и она не попадет вообще ни в один из выходных файлов SRS.

3. В терминале запустите:
   ```bash
   python3 scripts/compile_intersected.py
   ```
4. Скрипт **мгновенно** (благодаря кэшированию инвертированных списков в `cache/intersected_cache.json`) пересоберёт файлы в `output/`. 
   Вы увидите три файла: `intersected_all.srs`, `intersected_filtered.srs` (без альтернативных), и `intersected_alternative_only.srs` (только альтернативные маршруты). Общий `exclude.txt` будет исключён отовсюду.

### Как полностью локально пересобрать базы (если вышли обновления РКН)?
Если вы хотите полностью протестировать конвейер "как на GitHub", включая выкачивание свежих списков РКН и сообщества:
```bash
./pipeline.sh
```
Этот скрипт сделает `git pull` в папках зависимостей, **удалит ваш старый локальный кеш** `cache/intersected_cache.json` и прогонит полный цикл пересборки. Это занимает около нескольких секунд.

### Как добавить голосовые узлы Discord?
1. Откройте `scripts/fetch_tg_discord.py`.
2. Найдите массив `DISCORD_VOICE_IPS`.
3. Добавьте новые IP адреса голосовых серверов в этот массив.
4. Выполните: `python3 scripts/fetch_tg_discord.py`. Файл `output/tg_discord_cidrs.txt` будет обновлён.

### Как добавить новые сети (ASN), попавшие под 16KB блокировку?
Скрипт `scripts/fetch_asn_cidrs.py` автоматически качает JSON-файл c репозитория `Runnin4ik/dpi-detector`. 
Если нужно добавить новые AS-сети, просто обратитесь к автору того репозитория. Ваш скрипт автоматически подхватит изменения ночью через Actions. Если автор забыл вписать ASN, но вписал IP — скрипт самостоятельно сходит в RIPE API и динамически определит ASN.
