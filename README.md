# Smart RKN Routing для Sing-Box 🚀

Мощный и умный генератор правил маршрутизации для обхода блокировок РКН (Роскомнадзор) в клиентах на базе **Sing-Box**. 

Проект автоматически собирает актуальные заблокированные адреса и подсети, пересекает их с международными базами и ежедневно публикует обновлённые `.srs` и `.txt` файлы.

## 🤔 Почему не просто Re-filter-lists?

Существует отличный проект [Re-filter-lists](https://github.com/1andrevich/Re-filter-lists), который собирает реестр РКН и вычищает из него мусор (казино, порно, фейки). Однако, использование «голого» списка заблокированных доменов из реестра имеет фатальный недостаток: **фрагментация сервисов**.

Если РКН заблокировал условный домен `gateway.discord.gg`, то в сырых списках будет только этот домен. Но современный веб устроен сложнее: сервисы используют десятки поддоменов для API, телеметрии, CDN и авторизации. Если проксировать точечно один домен из реестра, сайт или приложение может открываться наполовину или не работать вообще.

**Решение — умное пересечение:**
1. Мы берем очищенную базу заблокированных доменов из `Re-filter-lists`.
2. Мы берем глобальную базу сайтов [domain-list-community](https://github.com/v2fly/domain-list-community) (где ресурсы аккуратно рассортированы по категориям: `discord`, `google`, `microsoft` и т.д.).
3. Скрипт ищет пересечения: если **хотя бы один** домен сервиса находится в блокировке РКН, скрипт вытягивает **всю категорию сервиса целиком** (со всеми его скрытыми поддоменами и API) и отправляет её в туннель.
4. (Важно!) Скрипт интеллектуально игнорирует общие категории (tld-ru, geolocation-!cn, category-ru), чтобы не завернуть в прокси "половину интернета", включая Госуслуги или Сбер.

## 📦 Дополнительные возможности

Помимо умного `.srs` для Sing-Box, проект генерирует:
* **`tg_discord_cidrs.txt`** — официальные подсети Telegram + IP-адреса голосовых серверов Discord.
* **`advanced_rkn_cidrs.txt`** — собранные через RIPE API подсети (CIDR) крупных провайдеров (Cloudflare, AWS, Hetzner, Akamai и др.), которые попали под ковровые блокировки (проблема 16 КБ DPI-фильтров).

## 🚀 Как использовать?

Клиенты VPN / Sing-Box могут использовать прямые ссылки из автогенерируемого раздела [Releases](../../releases/latest).

Доступно три варианта SRS правил:
- `intersected_filtered.srs` — Умная выжимка сервисов (без тех, что вы отсеяли в `alternative.txt`). Рекомендуется как основное правило для роутинга.
- `intersected_alternative_only.srs` — Содержит только сервисы из вашего `alternative.txt` (удобно для отдельного правила маршрутизации, например, направлять их через другой, более быстрый прокси).
- `intersected_all.srs` — Всё вместе в одном файле (без учета `alternative.txt`).

**Полное игнорирование (`exclude.txt`)**
Если вы впишете название категории (например, `google` или `microsoft`) в файл `exclude.txt`, этот сервис будет полностью и безвозвратно удален из **всех** скомпилированных файлов. Полезно, если сервис ложно срабатывает и вы хотите пускать его только напрямую.

Проект обновляется **автоматически каждую ночь** через GitHub Actions.
